## نظرة عامة على المشروع

بوت Telegram متقدم ومتكامل للتعامل مع خدمات SMM (Social Media Marketing)، تم تطويره بلغة Python مع ميزات احترافية تشمل:

- ✅ نظام رصيد متكامل للمستخدمين
- ✅ نظام تسعير مرن (نسبي أو ثابت)
- ✅ لوحة تحكم كاملة للأدمن
- ✅ حماية متقدمة من الطلبات المتعددة
- ✅ سجلات تفصيلية لجميع العمليات
- ✅ واجهة مستخدم سهلة ومتقدمة

---

## الميزات الرئيسية

### 1️⃣ نظام المستخدمين والرصيد
- إنشاء حسابات المستخدمين تلقائياً
- تخزين دائم للبيانات في ملف JSON
- دوال كاملة لإدارة الرصيد (إضافة، خصم، استعلام)
- منع الرصيد السالب
- تحديث فوري للرصيد

### 2️⃣ نظام التسعير المتقدم
- **التسعير النسبي (Percent)**: إضافة نسبة مئوية على السعر الأصلي
  - مثال: 50% على السعر الأصلي
- **التسعير الثابت (Fixed)**: إضافة مبلغ ثابت على السعر الأصلي
  - مثال: 5$ على كل طلب
- حساب تلقائي للأسعار
- عرض السعر الأصلي والسعر النهائي للمستخدم

### 3️⃣ نظام الأدمن (Single Admin)
- أدمن واحد فقط محدد بـ `ADMIN_ID`
- أوامر حصرية للأدمن:
  - إضافة/خصم رصيد المستخدمين
  - تعديل نظام التسعير
  - مراقبة العمليات
- رسائل خطأ للمستخدمين غير المصرح لهم

### 4️⃣ حماية متقدمة
- **قفل المستخدم (User Lock)**: منع الطلبات المتعددة في نفس الوقت
- **التحقق من المدخلات**: التأكد من صحة جميع البيانات
- **استرجاع تلقائي للرصيد**: إذا فشل الطلب يتم استرجاع المبلغ
- **معالجة الاستثناءات**: جميع الدوال محمية بـ try/catch

### 5️⃣ سجلات شاملة
- تسجيل جميع عمليات الرصيد
- تسجيل جميع الطلبات (ناجحة وفاشلة)
- تسجيل تغييرات التسعير
- تسجيل أوامر الأدمن
- تسجيل الأخطاء مع التفاصيل

## نظام التسعير

### أنواع التسعير

#### 1. التسعير النسبي (Percent)

يتم إضافة نسبة مئوية على السعر الأصلي.

**المعادلة:**
```
السعر النهائي = السعر الأصلي + (السعر الأصلي × النسبة ÷ 100)
```

**مثال:**
- السعر الأصلي: 10$
- النسبة: 50%
- السعر النهائي = 10 + (10 × 50 ÷ 100) = 15$

#### 2. التسعير الثابت (Fixed)

يتم إضافة مبلغ ثابت على السعر الأصلي.

**المعادلة:**
```
السعر النهائي = السعر الأصلي + المبلغ الثابت
```

**مثال:**
- السعر الأصلي: 10$
- المبلغ الثابت: 5$
- السعر النهائي = 10 + 5 = 15$

### ملف إعدادات التسعير

يتم تخزين الإعدادات في `pricing_config.json`:

```json
{
  "type": "percent",
  "value": 50
}
```

أو:

```json
{
  "type": "fixed",
  "value": 5
}
```

### حساب سعر الطلب

عند تقديم طلب، يتم حساب السعر الإجمالي كالتالي:

```python
# السعر لكل 1000 من API
base_rate = 10.0

# الكمية المطلوبة
quantity = 500

# السعر الأصلي للطلب
original_price = (base_rate × quantity) ÷ 1000
original_price = (10 × 500) ÷ 1000 = 5$

# السعر النهائي بعد تطبيق التسعير (مثال: نسبة 50%)
final_price = 5 + (5 × 50 ÷ 100) = 7.5$
```

## نظام الحماية والأمان

### 1. قفل المستخدم (User Lock)

**الغرض:** منع المستخدم من تقديم طلبات متعددة في نفس الوقت.

**كيفية العمل:**
- عند بدء طلب جديد، يتم قفل المستخدم
- مدة القفل الافتراضية: 60 ثانية
- إذا حاول المستخدم بدء طلب آخر أثناء القفل:
  ```
  ⏳ لديك طلب قيد المعالجة. يرجى الانتظار.
  ```
- يتم تحرير القفل تلقائياً بعد:
  - إكمال الطلب
  - إلغاء الطلب
  - فشل الطلب
  - انتهاء مدة القفل

**لتغيير مدة القفل:**  
في `config.py`:
```python
USER_LOCK_TIMEOUT = 60  # بالثواني
```

### 2. التحقق من المدخلات

**الرصيد:**
- يجب أن يكون رقماً
- يجب أن يكون موجباً
- يمنع الرصيد السالب

**الكمية:**
- يجب أن تكون رقماً صحيحاً
- يجب أن تكون موجبة
- يجب أن تكون أكبر من صفر

**الرابط:**
- يجب أن يبدأ بـ `http://` أو `https://`
- يمنع الروابط الفارغة أو غير الصحيحة

### 3. استرجاع الرصيد التلقائي

إذا فشل تقديم الطلب بعد خصم الرصيد:
1. يتم استرجاع المبلغ تلقائياً
2. يُخطر المستخدم باسترجاع الرصيد
3. يتم تسجيل العملية في السجلات

**رسالة الاسترجاع:**
```
❌ فشل تقديم الطلب.

تم استرجاع المبلغ إلى رصيدك تلقائياً.
يرجى المحاولة لاحقاً أو التواصل مع الدعم.
```

### 4. معالجة الاستثناءات

جميع الدوال محمية بـ `try/catch`:
- منع تعطل البوت
- تسجيل الأخطاء بالتفصيل
- إرسال رسائل خطأ واضحة للمستخدمين

## السجلات والتتبع

### ملف السجلات

جميع العمليات تُسجل في `log.txt` بصيغة:

```
[2026-04-07 15:30:45] [BALANCE] User:123456789 | add 10.5$ | Before: 5.0$ | After: 15.5$
[2026-04-07 15:31:20] [ORDER] User:123456789 | Order:456 | Service:123 | Qty:500 | Price:7.5$ | Status:success
[2026-04-07 15:32:10] [PRICING] User:ADMIN_ID | Admin:ADMIN_ID | Type:percent | Value:50
[2026-04-07 15:33:05] [ADMIN] User:ADMIN_ID | Command:/addbalance | Target:123456789 | Amount:10.5$
```

### أنواع السجلات

1. **BALANCE**: عمليات الرصيد
   - إضافة رصيد
   - خصم رصيد
   - استرجاع رصيد

2. **ORDER**: الطلبات
   - طلب ناجح
   - طلب فاشل
   - استرجاع بسبب فشل

3. **PRICING**: تغييرات التسعير
   - تغيير النسبة
   - تغيير المبلغ الثابت

4. **ADMIN**: أوامر الأدمن
   - جميع العمليات الإدارية

5. **ERROR**: الأخطاء
   - أخطاء النظام
   - أخطاء API
   - أخطاء المدخلات

### دوال التسجيل المتاحة

```python
# تسجيل عملية رصيد
logBalanceOperation(user_id, operation, amount, balance_before, balance_after)

# تسجيل طلب
logOrderOperation(user_id, order_id, service_id, quantity, total_price, status)

# تسجيل تغيير تسعير
logPricingOperation(admin_id, pricing_type, value)

# تسجيل أمر أدمن
logAdminOperation(admin_id, command, target_user, details)

# تسجيل خطأ
logErrorWithDetails(error_type, error_message, user_id, context)
```

## استكشاف الأخطاء

### المشكلة: البوت لا يستجيب

**الحلول:**
1. تحقق من أن البوت يعمل:
   ```bash
   python run.py
   ```

2. تحقق من `log.txt` للأخطاء

3. تأكد من صحة `BOT_TOKEN` في `config.py`

4. تحقق من اتصال الإنترنت

### المشكلة: خطأ في API

**الحلول:**
1. تحقق من `API_KEY` في `config.py`

2. تحقق من `API_URL` في `config.py`

3. اختبر API مباشرة:
   ```bash
   python test_api_direct.py
   ```

4. تحقق من السجلات في `log.txt`

### المشكلة: المستخدم لا يستطيع إضافة رصيد

**الحلول:**
1. تأكد من أن `ADMIN_ID` صحيح في `config.py`

2. تحقق من صيغة الأمر:
   ```
   /addbalance USER_ID AMOUNT
   ```

3. تحقق من السجلات لمعرفة سبب الفشل

### المشكلة: الطلب يفشل ويتم استرجاع الرصيد

**الحلول:**
1. تحقق من رصيدك في SMM API

2. تحقق من صحة الرابط المرسل

3. تحقق من أن الكمية ضمن الحدود المسموحة

4. راجع السجلات للتفاصيل

### المشكلة: الأسعار غير صحيحة

**الحلول:**
1. تحقق من إعدادات التسعير:
   ```
   /price
   ```

2. تأكد من نوع التسعير (percent أو fixed)

3. عدّل التسعير إذا لزم الأمر:
   ```
   /setpercent 50
   ```
   أو
   ```
   /setprice 5
   ```

### النسخ الاحتياطي

**ملفات مهمة للنسخ الاحتياطي:**
- `users_data.json` - بيانات المستخدمين والرصيد
- `pricing_config.json` - إعدادات التسعير
- `log.txt` - السجلات
- `config.py` - الإعدادات

**كيفية النسخ الاحتياطي:**
```bash
# إنشاء نسخة احتياطية
cp users_data.json users_data_backup.json
cp pricing_config.json pricing_config_backup.json
```

### تحديث البوت

1. أوقف البوت (Ctrl+C)

2. حدّث الملفات المطلوبة

3. شغّل البوت مرة أخرى:
   ```bash
   python run.py
   ```

### مراقبة السجلات

**عرض السجلات مباشرة:**
```bash
tail -f log.txt
```

**البحث عن أخطاء:**
```bash
grep "ERROR" log.txt
```

**البحث عن طلبات فاشلة:**
```bash
grep "Status:failed" log.txt
```

### تنظيف السجلات

إذا أصبح ملف `log.txt` كبيراً جداً:

```bash
# إنشاء نسخة احتياطية
cp log.txt log_backup.txt

# تفريغ الملف
> log.txt
```

## ملاحظات مهمة

### ⚠️ أمان

1. **لا تشارك `BOT_TOKEN` أو `API_KEY` مع أحد**
2. **احفظ `ADMIN_ID` بشكل صحيح**
3. **قم بنسخ احتياطي منتظم لبيانات المستخدمين**
4. **راقب السجلات بشكل دوري**

### 💡 نصائح

1. اختبر البوت قبل الاستخدام الفعلي
2. أضف رصيداً تجريبياً لنفسك أولاً
3. راجع السجلات بعد كل عملية مهمة
4. حدّث الأسعار بانتظام حسب السوق

### 🔧 التخصيص

يمكنك تعديل:
- رسالة الترحيب في `handle_start_command`
- معلومات التواصل في `handle_contact_admin`
- معلومات الدفع في `handle_payment_info`
- مدة قفل المستخدم في `config.py`
- أي نص آخر في `bot.py`
