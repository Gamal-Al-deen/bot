# 🚀 دليل النشر على Render - Deployment Guide

## 📋 المتطلبات المطلوبة

قبل تشغيل البوت، يجب عليك الحصول على:

1. **Telegram Bot Token** من @BotFather
2. **SMM API Key** من موقع SMM الخاص بك

---

## ⚙️ الإعداد الأولي

### 1. تعديل ملف config.py

افتح ملف `config.py` وقم بتعديل القيم التالية:

```python
# ⚠️ ضع هنا توكن البوت الخاص بك
BOT_TOKEN = "1234567890:ABCdefGHIjklMNOpqrsTUVwxyz"

# ⚠️ ضع هنا مفتاح API الخاص بموقع SMM
API_KEY = "your_smm_api_key_here"

# رابط API الخاص بموقع SMM
API_URL = "https://your-smm-site.com/api/v2"
```

### 2. التأكد من تثبيت Python

يجب أن يكون لديك Python 3.11 أو أحدث مثبتاً.

---

## 🖥️ التشغيل المحلي (للاختبار)

### 1. تثبيت المكتبات

```bash
cd bot
pip install -r requirements.txt
```

### 2. تشغيل البوت

```bash
python run.py
```

### 3. اختبار البوت

- افتح Telegram وابحث عن بوتك
- أرسل `/start` لترى رسالة الترحيب
- جرب الأزرار المختلفة

---

## ☁️ النشر على Render (مجاني)

### الخطوة 1: إنشاء حساب على Render

1. اذهب إلى https://render.com
2. سجل حساب مجاني
3. وصل حساب GitHub الخاص بك

### الخطوة 2: رفع الكود على GitHub

1. أنشئ repository جديد على GitHub
2. ارفع ملفات البوت

### الخطوة 3: إنشاء Web Service على Render

1. من Dashboard، اضغط **New +** → **Web Service**
2. اختر repository الذي رفعته
3. اضبط الإعدادات:

```
Name: smm-bot
Region: Frankfurt (أو الأقرب لك)
Branch: main
Root Directory: bot
Runtime: Python 3
Build Command: pip install -r requirements.txt
Start Command: python run.py
Instance Type: Free
```

4. اضغط **Create Web Service**

### الخطوة 4: الانتظار

- سيبدأ Render في بناء وتشغيل البوت
- انتظر 3-5 دقائق حتى يكتمل التشغيل
- عندما ترى "Running" باللون الأخضر، البوت جاهز!

---

## 🔍 مراقبة البوت

### عرض السجلات (Logs)

من Dashboard في Render:
1. اضغط على اسم البوت
2. اذهب إلى تبويب **Logs**
3. سترى جميع الأحداث والأخطاء

### ملف log.txt

البوت يسجل جميع الأحداث في `log.txt` تلقائياً.

---

## 🛡️ حل المشاكل الشائعة

### المشكلة: البوت لا يستجيب

**الحل:**
1. تحقق من أن BOT_TOKEN صحيح
2. تأكد من أن API_URL صحيح
3. راجع ملف `log.txt` للأخطاء

### المشكلة: خطأ في الاتصال بـ SMM API

**الحل:**
1. تحقق من API_KEY
2. تأكد من أن موقع SMM يعمل
3. راجع اللوغ للأخطاء التفصيلية

### المشكلة: البوت يتوقف على Render

**الحل:**
- Render Free قد يوقف البوت بعد 15 دقيقة من الخمول
- استخدم خدمة مثل UptimeRobot لإبقاء البوت نشطاً:
  1. سجل في https://uptimerobot.com
  2. أضف monitor جديد لنوع: HTTP(s)
  3. URL: `https://smm-bot.onrender.com/` (استبدل باسم بوتك)
  4. اضبط الفترة الزمنية لـ 5 دقائق

## 💡 نصائح إضافية

### لتحسين الأداء:

1. قلل عدد الخدمات المعروضة إذا كانت كثيرة
2. اضبط SLEEP_TIME إلى 1 بدلاً من 2 (في run.py)
3. استخدم caching للخدمات إذا أمكن

### للأمان:

1. أضف تحقق من معرف المستخدمين المسموح لهم
2. استخدم environment variables لـ BOT_TOKEN و API_KEY
3. لا ترفع ملف config.py على GitHub
